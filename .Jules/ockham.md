## 2025-05-26 - [Dead Code Removal] **Observation:** Found `tool.py` referencing non-existent modules (`src.jules_client`) and duplicate/transitional top-level service files (`src/gemini_client.py`, etc.) that were superseded by `src/services/`. This created confusion and potential security risks (`src/scraper.py` lacked SSRF protection). **Action:** Deleted `tool.py`, `src/gemini_client.py`, `src/github_client.py`, and `src/scraper.py`. Verified no usages in active codebase via grep and tests.
## 2025-05-27 - [CLI Refactor] **Observation:** Detected significant code duplication in `src/cli/commands.py`. The `handle_agent`, `handle_website`, and `handle_manual` functions all shared identical logic for printing idea summaries, executing workflows, and watching sessions. This violation of DRY increased the risk of inconsistent behavior if one handler was updated but others weren't. **Action:** Extracted the common logic into a helper function `_execute_and_watch(idea_data, args)`. This reduced code duplication and centralized the workflow execution path. Verified with tests.
